# csv_median_calculator (C++23)

`csv_median_calculator` — консольное приложение на **C++23** для обработки больших объемов биржевых данных. 

Приложение читает CSV-файлы с историей торгов из указанной директории, объединяет данные из всех подходящих файлов в единый поток, вычисляет медиану цен по мере обработки данных и сохраняет результаты в единственный выходной файл.

По мере чтения каждой новой цены из объединённого потока данных, программа пересчитывает медиану с учетом всех ранее прочитанных значений. Когда медиана изменяется — приложение фиксирует это в результирующем файле вместе с временной меткой и дополнительными метриками.

#### Основная функциональность:
- Рекурсивный поиск `level.csv`/`trade.csv` по маске в директории
- Парсинг `receive_ts;exchange_ts;price;...` → `MarketEvent` 
- **Лексикографическая сортировка** файлов + `stable_sort` по `receive_ts`
- **O(log N)** инкрементальная медиана через две кучи (max-heap + min-heap)
- Запись **только изменений** медианы (≥1e-8) с точностью 8 знаков

#### Дополнительная функциональность:
- Расширенные метрики через Boost.Accumulators: среднее, стандартное отклонение, квантили
- Гибкая конфигурация набора метрик через config.toml
- Поддержка квантилей p50, p90, p95, p99 через алгоритм extended_p_square
- Все дополнительные метрики выводятся в той же строке, что и изменение медианы

## Конфигурация приложения


### Формат конфигурационного файла

Конфигурация задается в формате TOML:


```toml
[main]
    # Путь к директории с входными CSV-файлами
    input = '/path/to/input_dir'
   
    # Путь к директории для выходных CSV-файлов
    # Если не указан, создается директория 'output' рядом с приложением
    output = '/path/to/output_dir'
   
    # Список масок имен файлов для фильтрации
    # Фильтрация работает как поиск подстроки: читаются только те файлы,
    # имена которых содержат хотя бы одну из указанных масок.
    # Если список пустой, читаются все CSV-файлы
    filename_mask = ['level', 'trade']

    # Список дополнительных метрик для вывода (доп. задание)
    # Поддерживаемые значения: "median", "mean", "std", "p50", "p90", "p95", "p99"
    # Порядок метрик в конфиге определяет порядок колонок в выходном файле
    # Если не указан — выводится только медиана
    metrics = ["median", "mean", "std", "p90", "p95", "p99"]
```

### Поддерживаемые метрики

| Значение в `metrics` | Колонка в CSV | Описание                              | Алгоритм                    |
|----------------------|---------------|---------------------------------------|-----------------------------|
| `"median"`           | `median`      | Медиана цен                           | Двухкучевой, точный O(log N)|
| `"mean"`             | `mean`        | Среднее арифметическое цен            | Boost.Accumulators          |
| `"std"`              | `std_dev`     | Стандартное отклонение цен            | Boost.Accumulators          |
| `"p50"`              | `p50`         | 50-й перцентиль (медиана, approx.)    | extended_p_square           |
| `"p90"`              | `p90`         | 90-й перцентиль                       | extended_p_square           |
| `"p95"`              | `p95`         | 95-й перцентиль                       | extended_p_square           |
| `"p99"`              | `p99`         | 99-й перцентиль                       | extended_p_square           |

**Примечание:** колонка `median` всегда рассчитывается точным двухкучевым алгоритмом (`MedianCalculator`) и используется для детектирования изменений. Квантили p50/p90/p95/p99 рассчитываются приближённо через `Boost.Accumulators::extended_p_square` и требуют достаточного объёма данных для сходимости.

## Входные данные

Данные из всех файлов, прошедших фильтр `filename_mask`, объединяются в единый набор и сортируются по `receive_ts`. Медиана рассчитывается по полю `price` из всех файлов. Разделитель колонок — точка с запятой (`;`).

#### Структура файла `level.csv`:
```csv
receive_ts;exchange_ts;price;quantity;side;rebuild
1716810808593627;1716810808574000;68480.00000000;10.10900000;bid;1
1716810808593627;1716810808574000;68479.90000000;0.00400000;bid;0
1716810808593627;1716810808574000;68480.20000000;4.52800000;ask;0
1716810808593627;1716810808574000;68480.40000000;0.10200000;ask;0
```

- `receive_ts` — временная метка получения данных (микросекунды)
- `exchange_ts` — временная метка биржи (микросекунды)
- `price` — цена
- `quantity` — объем
- `side` — сторона (`bid` или `ask`)
- `rebuild` — флаг перестроения стакана (0 или 1)


#### Структура файла `trade.csv`:

```csv
receive_ts;exchange_ts;price;quantity;side
1716810808663260;1716810808661000;68480.10000000;0.01100000;bid
1716810809314641;1716810809312000;68480.00000000;0.00200000;ask
1716810809719209;1716810809717000;68480.10000000;0.00500000;bid
1716810809719209;1716810809717000;68480.10000000;0.00200000;bid
1716810809719209;1716810809717000;68480.10000000;0.16900000;bid
```

- `receive_ts` — временная метка получения данных (микросекунды)
- `exchange_ts` — временная метка биржи (микросекунды)
- `price` — цена сделки
- `quantity` — объем сделки
- `side` — сторона (`bid` или `ask`)


## Выходные данные

Выходной файл — `median_result.csv`. Строка записывается **только при изменении медианы**. Изменением считается любое отклонение ≥ 1e-8.
При анализе одного файла `btcusdt_trade_2024.csv` результат:

#### Базовый режим (без `metrics` в конфиге):

```csv
receive_ts;price_median
1716810808663260;68480.10000000
1716810809314641;68480.05000000
1716810809719209;68480.10000000
```

#### Расширенный режим (`metrics = ["median", "mean", "std", "p90", "p95", "p99"]`):

```csv
receive_ts;median;mean;std_dev;p90;p95;p99
1716810808663260;68480.10000000;68480.10000000;0.00000000;0.00000000;0.00000000;0.00000000
1716810809314641;68480.05000000;68480.05000000;0.05000000;0.00000000;0.00000000;0.00000000
1716810809719209;68480.10000000;68480.06666667;0.04714045;0.00000000;0.00000000;0.00000000
```

- `receive_ts` — временная метка события, которое изменило медиану
- `median` — точное значение медианы (двухкучевой алгоритм)
- `mean` — среднее по всем ценам на момент записи
- `std_dev` — стандартное отклонение по всем ценам на момент записи
- `p90`, `p95`, `p99` — перцентили (сходятся на больших объёмах данных)

Дублирование `receive_ts` в выходном файле допустимо: если несколько записей с одинаковым `receive_ts` последовательно изменяют медиану, каждое изменение фиксируется отдельной строкой.
