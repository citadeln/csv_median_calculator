# csv_median_calculator (C++23)

`csv_median_calculator` — консольное приложение на **C++23** для обработки больших объемов биржевых данных. 

Приложение читает CSV-файлы с историей торгов из указанной директории, объединяет данные из всех подходящих файлов в единый поток, вычисляет медиану цен по мере обработки данных и сохраняет результаты в единственный выходной файл.

По мере чтения каждой новой цены из объединённого потока данных, программа пересчитывает медиану с учетом всех ранее прочитанных значений. Когда медиана изменяется — приложение фиксирует это в результирующем файле вместе с временной меткой.

#### Основная функциональность:
- Рекурсивный поиск `level.csv`/`trade.csv` по маске в директории
- Парсинг `receive_ts;exchange_ts;price;...` → `MarketEvent` 
- **Лексикографическая сортировка** файлов + `stable_sort` по `receive_ts`
- **O(log N)** инкрементальная медиана через две кучи (max-heap + min-heap)
- Запись **только изменений** медианы (≥1e-8) с точностью 8 знаков

## Требования

- Зависимости: CMake 3.23+, Boost 1.83+, git, make

## Конфигурация приложения


### Формат конфигурационного файла

Конфигурация задается в формате TOML:


```toml
[main]
    # Путь к директории с входными CSV-файлами
    input = '/path/to/input_dir'
   
    # Путь к директории для выходных CSV-файлов
    # Если не указан, создается директория 'output' рядом с приложением
    output = '/path/to/output_dir'
   
    # Список масок имен файлов для фильтрации
    # Фильтрация работает как **поиск подстроки**: читаются только те файлы, имена которых содержат хотя бы одну из указанных масок. Например, маска `'level'` соответствует файлу `btcusdt_level_2024.csv`, так как имя файла содержит подстроку `level`. Если список пустой, читаются все CSV-файлы
    filename_mask = ['level', 'trade']
```

## Входные данные

Данные из всех файлов, прошедших фильтр `filename_mask`, объединяются в единый набор и сортируются по `receive_ts`. Медиана рассчитывается по полю `price` из всех файлов. Разделитель колонок — точка с запятой (`;`).

#### Структура файла `level.csv`:
```csv
receive_ts;exchange_ts;price;quantity;side;rebuild
1716810808593627;1716810808574000;68480.00000000;10.10900000;bid;1
1716810808593627;1716810808574000;68479.90000000;0.00400000;bid;0
1716810808593627;1716810808574000;68480.20000000;4.52800000;ask;0
1716810808593627;1716810808574000;68480.40000000;0.10200000;ask;0
```

- `receive_ts` — временная метка получения данных (микросекунды)
- `exchange_ts` — временная метка биржи (микросекунды)
- `price` — цена
- `quantity` — объем
- `side` — сторона (`bid` или `ask`)
- `rebuild` — флаг перестроения стакана (0 или 1)


#### Структура файла `trade.csv`:

```csv
receive_ts;exchange_ts;price;quantity;side
1716810808663260;1716810808661000;68480.10000000;0.01100000;bid
1716810809314641;1716810809312000;68480.00000000;0.00200000;ask
1716810809719209;1716810809717000;68480.10000000;0.00500000;bid
1716810809719209;1716810809717000;68480.10000000;0.00200000;bid
1716810809719209;1716810809717000;68480.10000000;0.16900000;bid
```

- `receive_ts` — временная метка получения данных (микросекунды)
- `exchange_ts` — временная метка биржи (микросекунды)
- `price` — цена сделки
- `quantity` — объем сделки
- `side` — сторона (`bid` или `ask`)


## Выходные данные

Формат файла — две колонки, значение медианы выводится с точностью **8 знаков после запятой**:

```csv
receive_ts;price_median
1716810808663260;68480.10000000
1716810809314641;68480.05000000
1716810809719209;68480.10000000
```

- `receive_ts` — временная метка события, которое изменило медиану
- `price_median` — значение медианы, рассчитанное на основе всех цен, считанных до этого момента


В выходной файл записывается строка **только при изменении значения медианы**. Изменением считается любое отклонение с точностью до 8 знаков после запятой.

Дублирование `receive_ts` в выходном файле допустимо: если несколько записей с одинаковым `receive_ts` последовательно изменяют медиану, каждое изменение фиксируется отдельной строкой.
